# 다른 기기에서 전체 시나리오 검증 절차

운영 투입 전, **새 기기(개발 PC)** 에서 Deahyun 저장소를 clone 하여 LOAS 파이프라인
전체를 end-to-end 로 검증하는 절차다.  깊은 검증 항목·이미지 재로딩은 `시나리오검증.md`,
운영 배포 설정은 `운영서버_docker_설정.md` 를 참조한다.

> 이 기기 기준 절대경로(`/home/duckking/echopro/...`)가 일부 설정에 박혀 있으므로,
> **새 기기에선 [3. 기기별 경로 조정]을 반드시 수행**한다.

## 2. 저장소 clone (한 부모 폴더 아래, 폴더명 유지)

비공개(Deahyun) repo 이므로 GitHub 토큰이 필요하다.  토큰은 **`/home/duckking/echopro/.env`**
파일에서 읽는다(커밋 금지 시크릿).

```bash
# 1) .env 에서 토큰 추출
TOKEN=$(grep -oE 'ghp_[A-Za-z0-9]+' /home/duckking/echopro/.env | head -1)
[ -n "$TOKEN" ] || { echo "토큰을 .env 에서 찾지 못함"; }

# 2) 한 부모 폴더(~/svc) 아래로 4개 repo clone
mkdir -p ~/svc && cd ~/svc
for r in SocketDaim decision_agent PoolerTran Dumopro_Data_Analysis_WebApp; do
  git clone "https://${TOKEN}@github.com/Deahyun/${r}.git"
done

# 3) (선택) origin URL 에서 토큰 제거 — .git/config 에 토큰이 남는 걸 원치 않으면
for r in SocketDaim decision_agent PoolerTran Dumopro_Data_Analysis_WebApp; do
  git -C ~/svc/$r remote set-url origin "https://github.com/Deahyun/${r}.git"
done
# (3 을 적용하면 이후 git pull/push 시 토큰 URL 을 다시 써야 한다:
#   git -C ~/svc/<repo> pull "https://${TOKEN}@github.com/Deahyun/<repo>.git" main)
```
> ⚠️ **`SocketDaim` 폴더명을 유지**한다 — compose 가 만드는 네트워크 이름 `socketdaim_gw-net`
> 이 거기에 의존하고, decision_agent/PoolerTran/WebApp 이 external 로 그 이름에 붙는다.
> 이후 명령의 `~/svc` 는 본인 clone 경로로 바꿔 읽는다.
> ⚠️ 토큰은 시크릿이다 — **저장소/문서에 평문으로 커밋하지 말 것**(반드시 `.env` 에서 읽어 쓴다).

## 3. ★ 기기별 경로 조정 (필수)

저장소에 커밋된 로컬 override 에는 **이 기기 기준 절대경로**가 들어 있다.  새 기기 경로로 수정한다.

### 3-1. PoolerTran storage 마운트 경로
`PoolerTran/docker-compose.override.yml` 의 storage 경로를 **새 기기의 SocketDaim/storage 절대경로**로:
```yaml
    volumes:
      - /home/<나>/svc/SocketDaim/storage:/data/storage:ro   # ← 본인 절대경로로 수정
```
(ingestion-gw 가 영상을 쓰는 디렉터리와 **동일** 해야 PoolerTran 이 이미지를 읽는다.)

### 3-2. storage 디렉터리 생성 + 권한
ingestion-gw/cleaner 는 컨테이너 안에서 **uid 1000** 으로 실행된다.
```bash
cd ~/svc/SocketDaim && mkdir -p storage
# 본인 계정 uid 가 1000 이 아니면 컨테이너(uid1000)가 못 쓰므로 권한 개방:
id -u                      # 1000 이면 그대로 OK
sudo chown -R 1000:1000 storage   # 또는  chmod -R 777 storage
```

### 3-3. (확인) 로컬 override 가 켜져 있는지
clone 하면 아래가 함께 받아진다(폐쇄망 테스트용 — 이미 커밋됨):
- `SocketDaim/docker-compose.override.yml` (+ `loas_mariadb_init.sql`): mock LOAS MariaDB(daimresearch) + egress 접속
- `PoolerTran/docker-compose.override.yml`: `PT_REST_DEMO=true`, `PT_REST_DEMO_VERSION=2`, storage 경로
> `docker compose up`(dev) 시 자동 병합된다. 별도 작업 불필요(경로만 3-1 에서 수정).

## 4. (폐쇄망 전용) 이미지 사전 빌드 + 저장

폐쇄망에 반입할 이미지를 **미리 빌드해 tar 로 저장**하는 절이다.
> ⚠️ **인터넷 가능 환경에서 검증만 할 거면 이 절은 건너뛴다.** §5 의 `up -d --build`
> 가 필요 시 자동 빌드하므로, 여기서 또 빌드할 필요가 없다(중복 빌드 방지).

```bash
cd ~/svc
# 1) 빌드 + 베이스 이미지 확보
docker compose -f SocketDaim/docker-compose.yml                          build
docker compose -f decision_agent/docker-compose.yml                      build
docker compose -f PoolerTran/docker-compose.yml                          build
docker compose -f Dumopro_Data_Analysis_WebApp/docker/docker-compose.yml build
docker pull postgres:16 && docker pull redis:7-alpine && docker pull mariadb:11

# 2) 폐쇄망 반입용 tar (각 compose 가 쓰는 이미지 전부 = 앱 + 베이스)
mkdir -p ~/dist/images
docker save $(docker compose -f SocketDaim/docker-compose.yml config --images)                          | gzip > ~/dist/images/socketdaim-images.tar.gz
docker save $(docker compose -f decision_agent/docker-compose.yml config --images)                      | gzip > ~/dist/images/decision_agent-images.tar.gz
docker save $(docker compose -f PoolerTran/docker-compose.yml config --images)                          | gzip > ~/dist/images/poolertran-images.tar.gz
docker save $(docker compose -f Dumopro_Data_Analysis_WebApp/docker/docker-compose.yml config --images) | gzip > ~/dist/images/dumopro-images.tar.gz
```
> 폐쇄망에서는 위 tar 를 `docker load` 한 뒤 §5 를 **`--build` 없이**(`up -d`) 기동한다(빌드 불가).

## 11. 클린 재시작 (전부 삭제 후 재검증)

스키마/데이터를 깨끗이 초기화하고 처음부터 재검증할 때.
> ⚠️ **반드시 `down -v`(볼륨 삭제)** 여야 DB 스키마가 `init_db.sql` 로 새로 깔린다.
> `restart` 나 `-v` 없는 `up` 은 **구 볼륨이 남아 코드↔스키마 불일치**가 재발한다
> (예: admin_ui 9108 → `GET /admin/api/waypoints 500`,
> `function loas_station_id(integer) does not exist`).

```bash
# 1) 전부 삭제 (볼륨 포함, 의존 역순 — gw-net 소유자 SocketDaim 마지막)
cd ~/svc/Dumopro_Data_Analysis_WebApp && docker compose -f docker/docker-compose.yml down -v
cd ~/svc/PoolerTran                    && docker compose down -v
cd ~/svc/decision_agent                && docker compose down -v
cd ~/svc/SocketDaim                    && docker compose down -v && rm -rf storage/cctv
docker network rm socketdaim_gw-net 2>/dev/null || true

# 2) SocketDaim 먼저 (gw-net + gateway_db fresh = init_db 의 target_id 스키마 자동 적용)
cd ~/svc/SocketDaim
docker compose up -d --build postgres ingestion-gw egress-gw admin-ui loas-mariadb
until docker exec sd-postgres pg_isready -U postgres -d gateway_db >/dev/null 2>&1; do sleep 1; done

# 3) PoolerTran 큐 스키마 수동 적용 (init_db 에 없음 — cctv_transfer_queue/트리거/cctv_forwarder)
cd ~/svc/PoolerTran
docker exec -i sd-postgres psql -U postgres -d gateway_db \
    < migrations/migrate_010_cctv_transfer_queue.sql

# 4) 나머지 기동
cd ~/svc/decision_agent                && docker compose up -d --build
cd ~/svc/Dumopro_Data_Analysis_WebApp  && docker compose -f docker/docker-compose.yml up -d --build
cd ~/svc/PoolerTran                    && docker compose up -d --build
```
> fresh 볼륨이라 admin_ui(9108) target_id 화면이 정상 동작한다(스키마=코드 일치).
> 이후 **§6 송신 → §7 검증** 으로 재검증한다.

**데이터를 유지하며 스키마만 올리려면** (`down -v` 대신): 컨테이너는 두고 해당
마이그레이션만 적용한다. 예) 관측 개소 target_id 전환 →
```bash
cd ~/svc/SocketDaim
docker exec -i sd-postgres psql -U postgres -d gateway_db < scripts/migrate_010_station_by_target.sql
docker compose up -d --build admin-ui   # admin-ui 재기동
```

## 12. 폐쇄망 compose 파일 사용법 (base / override / deploy)

폐쇄망은 **빌드도 pull 도 불가**하므로(인터넷 없음), `up` 시 **어느 compose 파일을 쓰는지**가
핵심이다.  각 레포의 compose 파일 역할:

| 파일 | 역할 | 빌드 | `up` 시 자동 로드 |
|---|---|---|---|
| `docker-compose.yml` (base) | dev 기본. 자체빌드 서비스가 `build:` 를 가짐 | ✅ build | 항상 |
| `docker-compose.override.yml` | 로컬·테스트 추가분 — **SocketDaim**=mock `loas-mariadb`+egress 계정, **PoolerTran**=`PT_REST_DEMO`(더미 응답) | ✗ | **plain `up` 시 자동 병합** |
| `docker-compose.deploy.yml` | **폐쇄망 변종** — `build:` → `image: + pull_policy: never` (자체완결, 빌드/pull 안 함, 적재 이미지만 사용) | ✗ | 자동 아님(`-f` 로 지정) |

> 보유 현황: **4개 레포 모두 base+deploy 보유**(override 는 SocketDaim/PoolerTran 만).
> Dumopro 의 deploy 는 `docker/docker-compose.deploy.yml` 경로에 있다.

### 12-1. compose 병합 규칙 (반드시 이해)
- **plain `docker compose up`** → `base + override` **자동 병합**(= dev: mock/demo 포함, 필요 시 build).
- **`-f <파일>` 를 하나라도 주면 override 자동 로드가 꺼진다.** 지정한 파일만(여러 개면 왼→오 순서로 병합).
- 그래서 폐쇄망은 **`-f docker-compose.deploy.yml`** 로 띄우면 빌드/pull 없이 적재 이미지로만 기동되고,
  mock/demo(override)는 **필요할 때만 명시**해서 켠다.

### 12-2. 폐쇄망 기동 (이미지 `docker load` 이후, 빌드 없음)

**A) 운영 — 실제 LOAS/REST 사용 (mock·demo 제외)**
```bash
cd ~/svc/SocketDaim     && docker compose -f docker-compose.deploy.yml up -d
cd ~/svc/decision_agent && docker compose -f docker-compose.deploy.yml up -d
cd ~/svc/PoolerTran     && docker compose -f docker-compose.deploy.yml up -d
cd ~/svc/Dumopro_Data_Analysis_WebApp && docker compose -f docker/docker-compose.deploy.yml up -d
```
> 실제 LOAS 접속정보(egress)·실제 REST URL(PoolerTran)은 **env/시크릿으로 주입**(override 안 씀).

**B) 폐쇄망 테스트 — mock LOAS + 더미 REST (override 명시)**
```bash
cd ~/svc/SocketDaim && docker compose -f docker-compose.deploy.yml -f docker-compose.override.yml up -d
cd ~/svc/PoolerTran && docker compose -f docker-compose.deploy.yml -f docker-compose.override.yml up -d
cd ~/svc/decision_agent && docker compose -f docker-compose.deploy.yml up -d
cd ~/svc/Dumopro_Data_Analysis_WebApp && docker compose -f docker/docker-compose.deploy.yml up -d
```
> ⚠️ override 의 mock `loas-mariadb`(`image: mariadb:11`)도 빌드가 없으므로,
> **`mariadb:11` 이미지를 §4 에서 함께 `save`/`load`** 해 두어야 한다.

### 12-3. 순서·마이그레이션은 그대로
파일 선택(`-f docker-compose.deploy.yml`)만 바뀔 뿐, **기동 순서와 큐 마이그레이션은 §5·§11 과 동일**:
SocketDaim 먼저(gw-net+gateway_db) → `migrate_010_cctv_transfer_queue.sql` 수동 적용 → 나머지.
(`pgdata` fresh 볼륨이면 init_db 의 target_id 스키마가 자동 적용된다.)

> **요약**: 연결망 = plain `up`(자동 build+override) · 폐쇄망 = `-f docker-compose.deploy.yml`(빌드 없음, 적재 이미지).
> mock/demo 필요 시 `-f docker-compose.deploy.yml -f docker-compose.override.yml`.

## 14. 원격 웹 UI 접속 (SSH 터널)

폐쇄망 운영서버(`10.5.20.160`, SSH 포트 `50022`)의 웹 포트를 **내 PC localhost 동일 포트**로 우회한다.
`-L` 의 `localhost` 는 **서버 쪽 localhost**(= docker publish 포트)로 해석된다.

```bash
ssh -p 50022 -N -L 9108:localhost:9108 -L 9105:localhost:9105 -L 9107:localhost:9107 daim@10.5.20.160
```

연결 후 내 PC 브라우저에서:

| 포트 | URL | 화면 |
|---|---|---|
| 9108 | http://localhost:9108/ | SocketDaim 관리 (관측 개소 4 + 실시간 수신) |
| 9105 | http://localhost:9105/ | Dumopro 시각화 (Sync now → 데이터) |
| 9107 | http://localhost:9107/ | decision 판정 현황 + Threshold 패널 |

- **백그라운드**: `-N` → `-fN` 으로 바꿔 실행, 종료는 `pkill -f 'ssh -p 50022'`.
- **포그라운드(기본)**: 터널 유지, 종료는 `Ctrl-C`.
- **로컬 포트 충돌 시**: 왼쪽 숫자만 변경 — 예 `-L 19108:localhost:9108` → `http://localhost:19108/`.

> 페이지가 안 열리면 서버에서 `docker ps` 로 `sd-admin-ui`(9108)·`dumopro-api`(9105)·`da-decision-agent`(9107) 가 떠 있는지 먼저 확인한다.

## 15. REST 입·출력 확인 (PoolerTran)

PoolerTran 이 수신 서버로 보내는 **입력(payload)** 과 받은 **출력(body)** 을 로그로 확인한다.
`PT_API_LOGGING=true` 일 때 두 이벤트(`rest_api_request` / `rest_api_response`)가 찍힌다.

```bash
# 입출력 동시 추적(실시간)
docker logs -f poolertran 2>&1 | grep --line-buffered -E 'rest_api_request|rest_api_response'

# 입력만 / 출력만
docker logs poolertran 2>&1 | grep rest_api_request
docker logs poolertran 2>&1 | grep rest_api_response

# 보기 좋게(jq 있으면)
docker logs --since 10m poolertran 2>&1 | grep -E 'rest_api_request|rest_api_response' | jq .
```

- `rest_api_request` → **입력**: `payload`(`amr_id`, `target_id`, `frames[].received_time/file_path`)
- `rest_api_response` → **출력**: `status` + `body`(dual `[{score,path1,path2}(정적), {…}(동적)]`)
- 식별자 필드는 `target_id`(관측 개소 기준값).

**전제 1 — 로깅 ON 인지 확인** (`false` 면 안 찍힘):
```bash
docker exec poolertran env | grep PT_API_LOGGING
# false(deploy.yml 기본)면 잠깐 켜기:
#   PT_API_LOGGING:"true" 로 바꾼 뒤
#   cd ~/svc/PoolerTran && docker compose -f docker-compose.deploy.yml up -d --no-build --force-recreate poolertran
# (override 로 떠 있으면 이미 true)
```

**전제 2 — 트래픽 유발** (큐에 배치가 있어야 호출됨):
```bash
cd ~/svc/SocketDaim && python3 scripts/send_loas_scenario.py
```
