# 데모 → 실운영 전환 가이드 (폐쇄망 운영서버)

운영서버(`daim@daim-ecopro`, 소스 `~/svc/<레포>`)에서 현재 **데모 모드**로 돌고 있는 스택을
**실운영 모드**로 바꾸는 절차. 데모/실운영 차이는 한 줄로:

> **데모 = `docker compose up`(base+override 자동 병합)** — PoolerTran 더미 REST + SocketDaim 로컬 mock `loas-mariadb`
> **실운영 = `docker compose -f docker-compose.deploy.yml up`(override 제외)** — 외부 REST 프로그램 + 실제 LOAS MariaDB

---

## 1. 현재 데모 가동 명령 (지금 떠 있는 상태)

`docker compose ls` 기준, 현재는 다음으로 떠 있다(override 포함, 폐쇄망이라 `--no-build`):

```bash
# 1) SocketDaim  (docker-compose.yml + docker-compose.override.yml)
cd ~/svc/SocketDaim
docker compose up -d --no-build

# 2) decision_agent  (docker-compose.yml 단독 — override 없음)
cd ~/svc/decision_agent
docker compose up -d --no-build

# 3) PoolerTran  (docker-compose.yml + docker-compose.override.yml)
cd ~/svc/PoolerTran
docker compose up -d --no-build

# 4) Dumopro  (docker/docker-compose.yml 단독)
cd ~/svc/Dumopro_Data_Analysis_WebApp
docker compose -f docker/docker-compose.yml up -d --no-build
```

### 이때 override 가 만드는 "데모" 설정
| 레포 | override 가 주는 값(= 데모/테스트) |
|---|---|
| **PoolerTran** (`docker-compose.override.yml`) | `PT_REST_DEMO=true`(더미 dual 응답), `PT_REST_DEMO_VERSION=2`, `PT_API_LOGGING=true`(REST 입출력 로그) |
| **SocketDaim** (`docker-compose.override.yml`) | 로컬 **mock `loas-mariadb`** 컨테이너 추가 + egress 가 그 mock 으로 송신(`EGW_TARGET_DB_USER/PASSWORD` 주입) |
| decision_agent / Dumopro | override 없음 — 데모/실운영 차이 없음 |

> 즉 "데모"의 본질은 **① PoolerTran 이 외부 REST 대신 더미 응답**, **② egress 가 실제 LOAS 대신 로컬 mock DB** 로 쓰는 것.

## 2. 실운영 변경사항 (파일 수정)

> ⚠️ 실제 REST 수신은 **AnalysisReceiver 가 아니라 별도 외부 프로그램**이다. 주소는 `PT_REST_URL` 로 주입한다.
> 전송은 **경로(file_path) 방식**이라, 외부 프로그램이 그 경로의 이미지를 직접 읽을 수 있어야 한다(다른 머신이면 공유 스토리지 필요).

### 2-1. PoolerTran — `~/svc/PoolerTran/docker-compose.deploy.yml`
| 변수 | 데모 값 | 실운영 값 |
|---|---|---|
| `PT_REST_DEMO` | `"true"` | **`"false"`** |
| `PT_REST_URL` | `"http://analysis-receiver:8000/ingest"` | **`"http://<외부프로그램_호스트>:<포트>/<경로>"`** |
| `PT_PATH_REMAP_TO` | `/home/daim/svc/SocketDaim/storage` | 외부 프로그램이 접근하는 실제 경로(같은 서버면 그대로) |

```bash
sed -i \
  -e 's#PT_REST_DEMO: "true"#PT_REST_DEMO: "false"#' \
  -e 's#PT_REST_URL: ".*"#PT_REST_URL: "http://<외부프로그램_호스트>:<포트>/<경로>"#' \
  ~/svc/PoolerTran/docker-compose.deploy.yml
grep -nE 'PT_REST_DEMO:|PT_REST_URL:|PT_PATH_REMAP_TO:' ~/svc/PoolerTran/docker-compose.deploy.yml
```

### 2-2. SocketDaim — `~/svc/SocketDaim/docker-compose.deploy.yml` (egress 시크릿)
실운영은 로컬 mock 이 없으므로 egress 가 **실제 LOAS MariaDB(10.5.21.141)** 로 보낸다. placeholder 를 실제값으로:
| 변수 | 데모(override) | 실운영(deploy.yml) |
|---|---|---|
| `EGW_TARGET_DB_HOST` | `loas-mariadb`(로컬 mock) | **`10.5.21.141`** |
| `EGW_TARGET_DB_USER` | `daimresearch` | `daimresearch` |
| `EGW_TARGET_DB_PASSWORD` | `daimresearch1234!` | `daimresearch1234!` |

```bash
sed -i \
  -e 's#EGW_TARGET_DB_HOST: ".*"#EGW_TARGET_DB_HOST: "10.5.21.141"#' \
  -e 's#EGW_TARGET_DB_USER: ".*"#EGW_TARGET_DB_USER: "daimresearch"#' \
  -e 's#EGW_TARGET_DB_PASSWORD: ".*"#EGW_TARGET_DB_PASSWORD: "daimresearch1234!"#' \
  ~/svc/SocketDaim/docker-compose.deploy.yml
grep -nE 'EGW_TARGET_DB_(HOST|USER|PASSWORD):' ~/svc/SocketDaim/docker-compose.deploy.yml
```
> 이 비밀번호는 **서버 로컬 파일에만** 두고 GitHub 에 커밋하지 말 것.

### 2-3. decision_agent / Dumopro
변경 없음 — 그대로 deploy.yml 로 전환만 한다.

## 3. 실운영 가동 명령 (override 제외 = `-f deploy.yml`)

`-f docker-compose.deploy.yml` 을 주면 **override 가 자동 병합되지 않는다**. `--remove-orphans` 로 데모 전용 컨테이너(예: SocketDaim 의 mock `loas-mariadb`)도 정리된다.

```bash
# 1) SocketDaim — gw-net + gateway_db 생성 (반드시 먼저)
cd ~/svc/SocketDaim
docker compose -f docker-compose.deploy.yml up -d --no-build --remove-orphans

# 2) decision_agent
cd ~/svc/decision_agent
docker compose -f docker-compose.deploy.yml up -d --no-build --remove-orphans

# 3) PoolerTran  (위 2-1 수정 후)
cd ~/svc/PoolerTran
docker compose -f docker-compose.deploy.yml up -d --no-build --remove-orphans

# 4) Dumopro
cd ~/svc/Dumopro_Data_Analysis_WebApp
docker compose -f docker/docker-compose.deploy.yml up -d --no-build --remove-orphans
```

순서 주의: decision_agent·PoolerTran·Dumopro 의 deploy.yml 은 `socketdaim_gw-net` 을 **external** 로 참조 → SocketDaim 을 먼저 띄워야 한다.

## 4. 전환 후 검증

```bash
# (a) 모든 스택이 deploy.yml 하나씩으로 바뀌었는지
docker compose ls
#   → CONFIG FILES 가 전부 .../docker-compose.deploy.yml (콤마 없이 1개)

# (b) PoolerTran 이 외부로 실제 호출하는지 (demo 표시 없어야)
docker logs --since 5m poolertran 2>&1 | grep -E 'rest_api_request|rest_api_response'
#   → "demo": true 가 안 보이고 url/status 가 보이면 실운영

# (c) egress 가 실제 LOAS 로 INSERT 하는지
docker logs --since 5m sd-egress-gw 2>&1 | grep -E 'loas_inserted|loas_insert_failed'

# (d) 데모 전용 mock 컨테이너가 사라졌는지
docker ps --format '{{.Names}}' | grep loas-mariadb || echo "mock 제거됨(정상)"
```

체크리스트(실운영 정상):
- [ ] `docker compose ls` 의 CONFIG FILES 가 전부 deploy.yml 단독
- [ ] PoolerTran 로그에 `"demo": true` 없음 (외부 REST 실제 호출)
- [ ] 외부 REST 프로그램이 `file_path` 경로의 이미지를 읽을 수 있음(같은 서버/공유 스토리지)
- [ ] egress `loas_inserted` 발생(실제 LOAS 10.5.21.141)
- [ ] mock `loas-mariadb` 컨테이너 없음

## 5. 데모로 롤백

```bash
# 각 레포에서 override 포함(=‑f 없이) 으로 재기동
cd ~/svc/SocketDaim   && docker compose up -d --no-build
cd ~/svc/PoolerTran   && docker compose up -d --no-build
# decision_agent / Dumopro 는 base yml 그대로
```
→ override 가 다시 병합되어 PoolerTran 더미 REST + SocketDaim 로컬 mock 으로 복귀.

## 참고: 데모 ↔ 실운영 한눈 비교

| 항목 | 데모 | 실운영 |
|---|---|---|
| 기동 명령 | `docker compose up`(override 병합) | `docker compose -f docker-compose.deploy.yml up --remove-orphans` |
| PoolerTran REST | `PT_REST_DEMO=true` (더미) | `PT_REST_DEMO=false` + `PT_REST_URL`=외부 프로그램 |
| REST 수신 | 없음(더미 응답) | **외부 프로그램**(AnalysisReceiver 아님) |
| egress 대상 | 로컬 mock `loas-mariadb` | 실제 LOAS `10.5.21.141` |
| REST 로깅 | `PT_API_LOGGING=true` | `false` |
| storage 경로 | `/home/daim/svc/SocketDaim/storage` | 동일 |
